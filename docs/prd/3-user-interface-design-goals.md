# 3. User Interface Design Goals

## 3.1 Overall UX Vision (전체 UX 비전)

**"1초 만에 기록하고, 따뜻한 응원을 받는 경험"**

Drinky의 UX는 **최소한의 마찰**과 **긍정적 감정 설계**에 집중합니다. 기존 금주 앱들이 무겁고 진지한 톤으로 사용자에게 부담을 주었다면, Drinky는 가볍고 친근한 톤으로 매일 들어오고 싶은 앱을 지향합니다. 스트릭이 깨져도 좌절감보다는 "괜찮아요, 다시 시작해봐요!"라는 격려를 전달합니다.

## 3.2 Key Interaction Paradigms (핵심 인터랙션 패러다임)

- **원터치 체크:** 앱 진입 시 가장 먼저 보이는 큰 O/X 버튼, 한 번의 탭으로 기록 완료
- **즉각적 피드백:** 체크 시 애니메이션과 격려 메시지로 성취감 제공
- **점진적 공개:** 대시보드는 핵심 정보(스트릭, 오늘 상태)만 먼저 표시, 상세 차트는 스크롤/탭으로 접근
- **부드러운 전환:** 화면 간 자연스러운 트랜지션으로 앱 사용의 즐거움 강화

## 3.3 Core Screens and Views (핵심 화면)

| # | 화면 | 설명 |
|---|------|------|
| 1 | **온보딩 화면** | 앱 소개, 로그인/회원가입 |
| 2 | **메인 대시보드** | 오늘의 O/X 체크 버튼, 현재 스트릭, 간단한 주간 미리보기 |
| 3 | **기록 차트 화면** | 주간/월간 캘린더 뷰, 음주 패턴 그래프 |
| 4 | **설정 화면** | 알림 설정, 계정 관리, 앱 정보 |

## 3.4 Accessibility (접근성)

**WCAG AA 준수 권장**
- 충분한 색상 대비 (4.5:1 이상)
- 터치 타겟 최소 44x44px
- 스크린 리더 지원 (ARIA 레이블)

## 3.5 Branding (브랜딩)

- **톤앤매너:** 밝고 따뜻한, 친구처럼 격려하는 느낌
- **컬러 방향:**
  - Primary: 따뜻한 그린 또는 민트 계열 (건강, 긍정)
  - Accent: 부드러운 오렌지 또는 코랄 (응원, 에너지)
  - 다크 모드 고려
- **슬로건:** "더 오래, 더 함께"

## 3.6 Target Device and Platforms (타겟 기기 및 플랫폼)

**Web Responsive (PWA)**
- 모바일 웹 우선 (Mobile-first)
- iOS Safari, Android Chrome, Samsung Internet 최신 2개 버전 지원
- PWA로 홈 화면 추가 시 앱처럼 사용 가능
- 데스크톱은 MVP에서 낮은 우선순위

## 3.7 Primary User Flows (주요 사용자 흐름)

### Flow 1: 신규 사용자 온보딩
```
앱 접속 → 온보딩 슬라이드 → 회원가입/로그인 선택 →
이메일 또는 소셜 로그인 → 닉네임 설정 → 메인 대시보드
```

### Flow 2: 일일 체크인 (핵심 흐름)
```
앱 진입 → 대시보드 → O/X 버튼 탭 →
저장 완료 애니메이션 + 격려 메시지 → 스트릭 업데이트
```

### Flow 3: 기록 확인
```
대시보드 → 기록 탭 → 월간 캘린더 뷰 →
특정 날짜 탭 → 상세 정보 바텀시트
```

### Flow 4: 과거 기록 수정
```
기록 탭 → 수정할 날짜 탭 (7일 이내) →
수정 버튼 → O/X 변경 → 확인 다이얼로그 → 저장 → 스트릭 재계산
```

### Flow 5: 알림 설정
```
대시보드 → 설정 탭 → 알림 설정 →
ON/OFF 토글 또는 시간 변경 → 즉시 저장
```

### Edge Cases (엣지케이스)
- **스트릭 깨짐:** 부드러운 격려 메시지 + "다시 시작하기" 버튼
- **오프라인 체크인:** 로컬 저장 → 온라인 복귀 시 동기화
- **푸시 알림 거부:** 나중에 다시 묻기 옵션 제공
- **계정 삭제:** 확인 다이얼로그 2단계 + 30일 내 데이터 삭제
