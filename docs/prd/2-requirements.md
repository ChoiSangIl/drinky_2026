# 2. Requirements

## 2.1 Functional Requirements (기능 요구사항)

| 우선순위 | ID | 요구사항 |
|---------|-----|---------|
| **P0 (필수)** | FR1 | 사용자는 이메일 또는 소셜 로그인(Google/Kakao)으로 계정을 생성하고 로그인할 수 있다 |
| **P0 (필수)** | FR2 | 사용자는 오늘 음주 여부를 O(마심)/X(안마심) 원터치로 기록할 수 있다 |
| **P0 (필수)** | FR3 | 시스템은 연속으로 술을 마시지 않은 날을 자동으로 카운트하여 스트릭으로 표시한다 |
| **P1 (중요)** | FR4 | 사용자는 주간/월간 음주 기록을 차트(캘린더 뷰 또는 그래프)로 확인할 수 있다 |
| **P1 (중요)** | FR5 | 시스템은 설정된 시간에 격려 푸시 알림을 발송한다 |
| **P1 (중요)** | FR6 | 사용자는 푸시 알림 수신 여부와 시간을 설정할 수 있다 |
| **P2 (권장)** | FR7 | 사용자는 자신의 스트릭 기록과 총 금주 일수를 대시보드에서 확인할 수 있다 |
| **P2 (권장)** | FR8 | 사용자는 과거 날짜의 음주 기록을 수정할 수 있다 (최대 7일 이내) |

## 2.2 Non-Functional Requirements (비기능 요구사항)

- **NFR1:** 앱 첫 로딩 시간은 3초 이내여야 한다
- **NFR2:** 음주 체크 인터랙션은 100ms 이내에 반응해야 한다
- **NFR3:** PWA로 구현하여 모바일 웹에서 앱처럼 사용 가능해야 한다
- **NFR4:** Chrome, Safari, Samsung Internet 최신 2개 버전을 지원해야 한다
- **NFR5:** 음주 기록은 민감 정보로 취급하여 암호화하여 저장해야 한다
- **NFR6:** 개인정보는 최소한으로 수집한다 (이메일, 닉네임 정도)
- **NFR7:** 한국어 UI를 기본으로 제공한다 (다국어는 MVP 이후)
